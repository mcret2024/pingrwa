⚠ 운영 전환 체크리스트 (클릭하여 펼치기)
🔐 보안 (Critical)
  • JWT_SECRET / ADMIN_JWT_SECRET 재발급 — devnet 의 secret 그대로 두면 다른 환경에서 발급된 토큰 통용 가능. 32+자 랜덤.
  • CRON_KEY 재발급 — Hostinger cron URL 의 ?key= 와 .env 의 값 모두 새로 발급. devnet key 노출 위험 차단.
  • ADMIN_KEY 재발급 — admin API 호출용 키. .env 의 값 갱신.
  • ADMIN_WALLET_ADDRESSES 설정 (v583) — 관리자 Phantom 지갑 화이트리스트. 쉼표 구분 다중 등록. 비워두면 화이트리스트 비활성 — 운영 환경에서는 반드시 채워두기.
🛠 UI / 운영 도구
  • 🔴 [BLOCKER] testInterestBtn 코드 완전 제거 (운영 전환 차단 조건)
    devnet 회귀 테스트용 버튼. force-interest 의 mode='test' 가 중복 체크를 우회하므로, mainnet 에서 운영자가 실수로 누르면 사용자에게 진짜 USDT 더블 지급이 발생합니다.

    운영 전환 전 반드시 제거:
    • admin/staking.html — <button id="testInterestBtn"> 블록 및 testInterestCard 제거
    • admin/js/staking.js — testInterestBtn 관련 변수 선언/이벤트 등록 제거
    • admin/staking.html 페이지에서 'testInterestBtn' 텍스트 검색 → 0건 확인
    • (선택) php-api/routes/staking.php 의 force-interest 엔드포인트에서 mode='test' 분기 코드 제거 — UI 호출자가 없으면 무해하지만 dead code 정리 차원

    미제거 시 위험: force-interest 의 mode='disaster' (기본) 는 per-holder 중복 체크가 있어 안전하지만, mode='test' 는 의도적으로 중복 체크 우회 → 같은 회차에 두 번 클릭하면 모든 사용자가 두 번 지급됨.
